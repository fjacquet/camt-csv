  <action>
Create two formatters:

**StandardFormatter (internal/formatter/standard.go)**:
- Header() returns 35 columns matching current output: BookkeepingNumber, Status, Date, ValueDate, Name, PartyName, PartyIBAN, Description, RemittanceInfo, Amount, CreditDebit, IsDebit, Debit, Credit, Currency, AmountExclTax, AmountTax, TaxRate, Recipient, InvestmentType, Number, Category, Type, Fund, NumberOfShares, Fees, IBAN, EntryReference, Reference, AccountServicer, BankTxCode, OriginalCurrency, OriginalAmount, ExchangeRate
- Format() calls tx.MarshalCSV() for each transaction (preserves existing behavior)
- Delimiter() returns ',' (comma)
- No date format parameter yet (Task 3 will add configurable dates)

**iComptaFormatter (internal/formatter/icompta.go)**:
- Header() returns 10 columns: Date, Name, Amount, Description, Status, Category, SplitAmount, SplitAmountExclTax, SplitTaxRate, Type
- Format() projects Transaction fields:
  - Date: tx.Date.Format("02.01.2006") (dd.MM.yyyy per research)
  - Name: tx.Name (or tx.PartyName if Name empty)
  - Amount: tx.Amount.StringFixed(2)
  - Description: tx.Description
  - Status: map CAMT statuses to iCompta (BOOK/RCVD→"cleared", PDNG→"pending", REVD/CANC→"reverted", default→"cleared")
  - Category: tx.Category (warn if empty, use "Uncategorized" as fallback)
  - SplitAmount: tx.Amount.StringFixed(2) (same as Amount for v1)
  - SplitAmountExclTax: tx.AmountExclTax.StringFixed(2)
  - SplitTaxRate: tx.TaxRate.StringFixed(2)
  - Type: tx.Type
- Delimiter() returns ';' (semicolon per iCompta plugins)
- Log warning for transactions with missing category

Both formatters:
- Implement OutputFormatter interface
- Use decimal.StringFixed(2) for amounts (never float64)
- Handle zero dates gracefully (return empty string)
- Are stateless (no mutable fields)

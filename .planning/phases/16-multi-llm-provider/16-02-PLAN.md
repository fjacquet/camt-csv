1. Check apiKey empty — log debug, set transaction.Category = models.CategoryUncategorized, return transaction, nil
2. Check rate limiter — return error if exceeded
3. Build prompt using same buildCategorizationPrompt logic (copy the method or call a shared helper — see note below)
4. Call callAPIWithRetry() — same retry pattern as GeminiClient.callGeminiAPIWithRetry()
5. Apply cleanCategory() — copy the method from GeminiClient (same logic needed)
6. Set transaction.Category and return

Note on shared prompt/clean logic: To avoid DRY violations, the prompt and cleanCategory methods can be copied into openrouter_client.go for now (they are standalone functions with no receiver dependency on GeminiClient state). The executor may also extract them to a shared categorizationHelpers.go file — use judgment based on what keeps the code cleanest with minimal diff.

callAPI() implementation:

- POST to {c.baseURL}/chat/completions
- Body: OpenRouterRequest{Model: c.model, Messages: [{Role: "user", Content: prompt}]}
- Header: Authorization: Bearer {c.apiKey}
- Header: Content-Type: application/json
- Parse OpenRouterResponse, extract choices[0].message.content
- Return strings.TrimSpace(content), nil

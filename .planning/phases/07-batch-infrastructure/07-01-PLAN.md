Create `internal/batch/processor.go` with:

**Structure:**
```go
// BatchProcessor handles standardized batch processing for any parser
type BatchProcessor struct {
    parser   parser.FullParser
    logger   logging.Logger
}

func NewBatchProcessor(p parser.FullParser, logger logging.Logger) *BatchProcessor
```

**Core method:**
```go
// ProcessDirectory processes all files in inputDir, writes to outputDir
// Returns manifest (never nil) and error only for config/permission issues
// Individual file failures are captured in manifest, not returned as errors
func (bp *BatchProcessor) ProcessDirectory(ctx context.Context, inputDir, outputDir string) (*BatchManifest, error)
```

**Implementation requirements:**
1. **File discovery:** Use `filepath.Walk` to find all files in `inputDir` (not recursive, only top-level)
2. **File filtering:** Skip hidden files (starting with `.`), only process files matching parser's expected extension
3. **Sequential processing:** Process files one-by-one in alphabetical order (no parallel processing for Phase 7)
4. **Error isolation:** Each file's validation/parse/write errors are captured in `BatchResult.Error`, never stop batch
5. **Per-file processing:**
   - Validate format using `parser.ValidateFormat(filePath)`
   - If invalid, set `result.Error = "validation_failed"` and continue
   - Open file, call `parser.Parse(ctx, file)`
   - If parse error, set `result.Error = err.Error()` and continue
   - Generate output filename (preserve input basename, change extension to `.csv`)
   - Write CSV using `parser.WriteToCSV(transactions, outputPath)`
   - If write error, set `result.Error = fmt.Sprintf("write_error: %v", err)` and continue
   - If success, populate `result.RecordCount = len(transactions)`
6. **Timing:** Record start time at beginning, calculate duration at end
7. **Always write manifest** to `{outputDir}/.manifest.json` before returning

**Helper methods (private):**
- `discoverFiles(inputDir string) []string` — Returns sorted list of processable files
- `processFile(ctx context.Context, filePath, outputDir string) BatchResult` — Processes single file, returns result (never panics)

**Error handling:**
- Return error only if `inputDir` doesn't exist or `outputDir` can't be created
- All file-level errors go into manifest results, not function error return
- Log warnings for skipped files (invalid format, read errors) using logger

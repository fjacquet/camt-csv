Rewrite `RunConvert` in `cmd/common/convert.go` to implement the following logic:

```
if fileInfo.IsDir():
    if outputPath == "":
        logger.Fatalf("--output flag is required when input is a folder: specify the output directory with -o or --output")
    FolderConvert(ctx, p, inputPath, outputPath, logger, format, dateFormat)
else:
    ProcessFile(...)  // unchanged
```

Add a new exported function `FolderConvert` that replaces `BatchConvertLegacy` for CAMT/debit/selma/revolut-investment parsers. `FolderConvert` must:
1. Resolve the formatter from `formatter.NewFormatterRegistry().Get(format)` — exit with error on invalid format
2. Assert the parser to `parser.FullParser` — exit if not supported
3. Create `batch.NewBatchProcessor(fullParser, logger, outFormatter)`
4. Call `processor.ProcessDirectory(ctx, inputDir, outputDir)`
5. Write the manifest via `manifest.WriteManifest(filepath.Join(outputDir, ".manifest.json"))`
6. Log `"Batch complete: N/M files succeeded"` (reuse existing log pattern from revolut/convert.go)
7. Call `os.Exit(manifest.ExitCode())` only if exit code != 0

Keep `BatchConvertLegacy` in the file (not deleted) — it is still used by camt/debit/selma/revolut-investment via the old `BatchConvert` interface, but `RunConvert` will now call `FolderConvert` instead. `BatchConvertLegacy` can be left unexported or kept as-is; remove its usage from `RunConvert` only.

Import additions needed: `"fjacquet/camt-csv/internal/formatter"`, `"fjacquet/camt-csv/internal/parser"` (already imported in process.go but may be needed here too).

Do NOT remove the `BatchConvertLegacy` function itself — it may be referenced by older code paths and will be removed in Phase 13 with the full batch cleanup.

Error message for missing `--output` must be human-readable, example:
`"--output flag is required when processing a folder. Use -o or --output to specify the output directory."`

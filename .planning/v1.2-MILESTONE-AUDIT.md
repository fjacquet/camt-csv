## Cross-Phase Integration Gaps

### Gap 1: Revolut Batch Uses Legacy Code (Medium)
- **Location:** `internal/revolutparser/adapter.go:69-70`
- **Issue:** `BatchConvert()` delegates to standalone `BatchConvert()` function instead of `BatchProcessor`
- **Impact:** No manifest.json, no semantic exit codes in Revolut batch operations
- **Fix:** Refactor adapter to use `batch.NewBatchProcessor(a, a.GetLogger())`

### Gap 2: Batch/Consolidation Paths Ignore Formatters (Medium)
- **Location:** `cmd/pdf/convert.go:96`, `cmd/pdf/convert.go:100-228`
- **Issue:** `pdfBatchConvert()` and `consolidatePDFDirectory()` don't pass format flags
- **Impact:** --format flag silently ignored in directory modes
- **Fix:** Pass format/dateFormat through batch and consolidation functions

### Gap 3: BatchProcessor Missing Formatter API (Low)
- **Location:** `internal/batch/processor.go`
- **Issue:** ProcessDirectory doesn't support formatter configuration
- **Impact:** BatchProcessor outputs standard format only
- **Fix:** Add formatter options to BatchProcessor or process files through formatted pipeline

## Summary

Removed 6 redundant/dead fields from the 35-column standard CSV format, reducing it to 29 columns. The iCompta formatter remained unchanged at 10 columns.

### Key Accomplishments

1. **StandardFormatter trimmed to 29 columns** — Removed BookkeepingNumber, IsDebit, Debit, Credit, Recipient, AmountTax
2. **MarshalCSV/UnmarshalCSV updated** — Transaction serialization reduced from 35 to 29 fields
3. **All parser tests aligned** — CAMT, PDF, Selma, Revolut, Revolut Investment, Debit all pass with 29-column format
4. **End-to-end tests added** — New integration tests verify both standard (29-col) and iCompta (10-col) output
5. **Hardcoded header fixed** — common/csv.go hardcoded header updated from 35 to 29 columns

### Removed Fields

| Field | Reason |
|-------|--------|
| BookkeepingNumber | Never populated by any parser |
| IsDebit/DebitFlag | Redundant with CreditDebit field |
| Debit | Derived from Amount + CreditDebit |
| Credit | Derived from Amount + CreditDebit |
| Recipient | Redundant with Name/PartyName |
| AmountTax | Never populated by any parser |
